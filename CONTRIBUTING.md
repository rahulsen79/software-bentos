# Contributing (no IDE needed)

Everything here is markdown, and you can add or edit it entirely in your browser on
github.com. No install, no git commands, no IDE.

## Edit an existing file

1. Open the file on github.com.
2. Click the pencil icon (Edit this file), top right of the file.
3. Make your changes in the browser.
4. Click **Commit changes**. Choose "Create a new branch and start a pull request,"
   then **Propose changes**. (If you have write access, you can commit straight to main.)
5. The owner reviews and merges. Done.

## Add a new skill or instruction

1. Open the right folder (for example `figma-make/skills/`).
2. See example.md or README for suggested template
3. Click **Add file**, then **Create new file**.
4. Name it, for example `generate-empty-states.md`.
5. Paste your content. Use the template below.
6. Click **Commit changes** and propose it, same as above.

Already wrote the file somewhere else? Use **Add file, Upload files** and drag it in.

## Even easier: the web editor

Open the repo and press the `.` key (or change `github.com` to `github.dev` in the URL).
This opens a full editor in your browser, still no install, where you can edit several
files at once. Commit from the Source Control panel on the left.

## The one rule that keeps this folder healthy

Add something here only when you have had to retype the same thing twice. A one-off prompt
stays in your chat. A recurring one becomes a file everyone can reuse.
